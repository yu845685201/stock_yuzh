# 项目概述

## 项目名称
A股盘后静态分析系统

## 项目简介
本项目是一个专注于A股市场的盘后静态数据分析系统，主要用于交易日结束后的数据分析和研究。系统包含两大核心模块：**数据同步**和**股票分析**，为投资决策提供数据支持。

---

# 核心原则

## 数据管理原则
1. **数据真实性**
   - 严禁使用模拟数据(mock data)或伪造数据
   - 所有数据必须来自真实可靠的源

2. **数据完整性处理**
   - 能够直接同步的数据，如实同步
   - 无法直接同步但可计算的数据，通过计算获得
   - 无法获取或计算的数据，留空处理

## 技术架构原则
1. **基础技术栈**
   - 后端语言：Python 3.8+
   - 数据库：PostgreSQL 12+
   - 采用面向接口的编程范式

2. **数据源管理**
   - 支持多数据源接入（目前支持pytdx、baostock）
   - 设计统一的数据接口规范
   - 支持数据源的热插拔

3. **系统设计**
   - 优先考虑扩展性和可维护性
   - 采用模块化设计，功能模块高内聚低耦合
   - 便于后续新增数据源和分析功能

# 开发与部署规范

## 开发准则
1. **数据源合规性**
   - 严格按照产品设计文档指定的数据源采集数据
   - 任何数据源变更必须经过评审并更新文档

2. **环境隔离**
   - 严格区分测试环境(UAT)和生产环境(PROD)
   - 严禁跨环境使用目录或数据库资源
   - 环境配置通过配置文件管理，禁止硬编码

3. **目录管理**
   - 严格按照规定的目录结构组织文件
   - 禁止在未授权的情况下创建新的顶级目录
   - 各目录用途明确，禁止混用
